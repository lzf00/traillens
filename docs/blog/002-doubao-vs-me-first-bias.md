> 这是 TrailLens build-in-public 第 2 篇。今天我做了一件小事:让豆包(火山方舟)
> 给我的 5 张风光照打 8 维分数,然后我自己再校准。
> 一组真实的数据告诉我:**我和 AI 的整体审美对齐 ±0.36 — 比想象的近**,
> 但有 2 张照片显示出"我的审美偏好"AI 看不到的细节。
> 这正是 PIAA(个人美学)训练的意义。

---

## 一句话结论

豆包给我的 5 张风光照的 overall 评分,和我自己的真实校准对比:

```
      豆包         我
  #1  8.3  →  9.0   ↑ +0.7   (豆包低估了我的"心头好")
  #2  7.8  →  7.5   ↓ -0.3
  #3  7.8  →  8.0   ↑ +0.2
  #4  7.1  →  6.5   ↓ -0.6   (豆包高估了 — 我更挑剔)
  #5  7.5  =  7.5         (完全一致 — 共同基准)
  ──────────────────
  平均带符号偏差:  +0.00      ← 没有系统性偏向
  平均绝对偏差:    ±0.36      ← 比预期的小
```

## 这意味着什么

**3 个发现:**

**1. 豆包对风光照"够用"** — 平均带符号偏差是 0,意味着它没有系统性高估或低估我的所有照片。
   平均绝对偏差仅 0.36 分(满分 10),远小于 1.0 这个"显著差异"门槛。

**2. 但单张照片有审美偏差** — 这正是个性化(PIAA)训练的意义。
   - #1 那张是我的"心头好"豆包没看出来。
   - #4 那张豆包给了我觉得过高的分。
   - 学习这些差异 = 训出一个**理解我审美**的 LoRA。

**3. 5 张数据完全不够训** — 但 5 张数据**够验证整套流水线**。
   下一步是收集 50-200 张照片,让差异有统计意义。

## 我用了什么栈

(对工程同行可能感兴趣)

- **照片预打分**: 豆包(火山方舟)`doubao-seed-2-0-pro-260215` Responses API,
  每张 ¥0.02。Thinking 模型,先 reasoning 再输出 8 维 JSON。
- **本地标注 UI**: 100 行 Python 标准库 http.server + 单文件 HTML,
  0 依赖。用户按 1-9 校准。
- **偏差分析**: 跨 8 维(构图/视觉/技术/原创/主题/情感/格式塔/综合)。

代码全开源在 [`packages/annotation/`](https://github.com/lzf00/traillens)。
